Critique and Iteration

We presented our hifi mockups to an industry critic and our peers in our user interface and user experience class to receive feedback on the designs and prototypes that we had created. This was really useful in helping us improve our designs because it allowed us to understand how new users would interact with and interpret our designs.

Some key points of feedback that we received and our solutions to those issues:

The effect of the “Cancel” button is unclear: does it close the appointment popup or cancel the appointment itself?
“Cancel” was rephrased to “Cancel Appointment”, and added an “X” on the top right of the appointment popup to close the popup.
Scheduling appointments is a key feature, and should also be accessible from the home page. Redundancy is ok in this context.
A “Schedule Appointments” button was added directly to the home page, in a highly visible space.
Finding new therapists is crucial to the app’s mission, and should be made more accessible.
A “Search” tab was added to the bottom navbar. Clicking on “Search” brought users to a page where exploring and matching with new therapists is the main functionality (as opposed to scheduling an appointment, which is the main functionality of the “Schedule” tab).
When new messages are received, a notification signal on the messages tab in the bottom navbar would be helpful.
When there are unread messages, a notification icon appears on the “Messages” tab in the bottom navbar to bring attention to new messages.
A therapist’s profile should be accessible both from the chat function, and from the “your primary therapist” section in the profile page.

In a chat screen, an “information” icon was added besides the therapist’s profile icon that brings the user to the therapist’s full profile.

In the profile page, “your primary therapist” is now displayed as a card (in the same format as search results), and clicking it brings the user to the therapist’s full profile.

Missing an option to set someone as your primary therapist.
A button was added to the therapist profile page to set the current therapist as the user’s primary therapist.
Inconsistencies in data (eg. names, dates, times) across different screens.
The prototype was re-organized, and any inconsistencies in information were fixed.

Finally we combined all of this feedback and new ideas and incorporated them into an improved version of our prototype.

Conclusion

This project was a great chance to apply UIUX principles in a real-world context, and our team had a lot of fun throughout the design process. The group format was an amazing opportunity to both collaborate with fellow UIUX students while learning from each other; moreover, the fact that we were basing our designs off a real startup made our project feel that much more impactful. Overall, this project was a great exercise in design, teamwork, and creativity.
